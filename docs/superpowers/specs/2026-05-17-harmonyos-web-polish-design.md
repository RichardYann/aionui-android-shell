# HarmonyOS Web 壳安卓一致化设计

## 背景

- 当前 Harmony Web 壳已经能联网并能正常打开网页，但仍存在明显的体验差异。
- 用户确认希望优先采用“安卓一致化”方向，而不是保守优化或顶栏常驻。
- 这轮优化的目标不是新增功能，而是把已有 Web 壳的加载反馈、导航召回和品牌视觉拉回到更接近 Android 版本的体验。

## 目标

- 减少进入页面后“先卡一下再出现进度条”的空窗感。
- 页面首屏内容出现后，立即移除中央加载遮罩，不再继续挡住网页。
- 去掉顶部“半透明网页标题栏”的观感，只保留纯操作型导航条。
- 让顶部导航隐藏后的下滑召出方式更接近 Android 版本。
- 将 HarmonyOS 应用图标替换为与 Android 一致的 AionUi 黑白图标。

## 非目标

- 本次不调整 ConnectPage 的结构与样式。
- 本次不重做切换服务器弹层布局。
- 本次不引入新的品牌样式或重新设计 Android 图标。
- 本次不继续保留运行时调试信息在正式包中显示。

## 现状对照

### 加载反馈

- Harmony 目前同时使用顶部进度条和中央加载遮罩。
- 中央遮罩跟随 `pageLoading` 结束，而 `pageLoading` 主要依赖 `onPageEnd` 才关闭。
- 这导致两个问题：
  - 刚进入页面到 `onPageBegin` 触发前有空窗，用户感觉“先卡一下”。
  - 页面内容已经可见后，中央转圈仍会再停留一小段时间。

### 顶部导航

- Harmony 当前顶部导航是深色块状工具条，加上之前的调试文字浮层，视觉上容易被理解为“网页标题栏”。
- Android 版本更接近纯操作条，没有额外半透明标题感。

### 导航召回

- Android 版本通过顶部一条很薄的热区配合向下滑动手势召回导航，阈值较容易触发。
- Harmony 当前隐藏态热区较弱，召回时机和可操作性都不如 Android。

### 图标

- Android 图标是黑底白线条图形。
- Harmony 图标目前是蓝色渐变 A，品牌识别不一致。

## 选定方案

采用“安卓一致化”方案。

核心原则：

- 保留 Harmony 当前已有的功能结构。
- 优先调整加载状态机、导航显示逻辑和资源文件。
- 所有变化都以“更接近 Android 当前交互”为目标，而不是做 Harmony 独立视觉风格。

## 设计细节

### 1. 加载状态收口

将当前单一 `pageLoading` 拆成更贴近用户感知的两个阶段：

- `bootLoading`
  - 表示页面已进入 Web 壳，但 ArkWeb 还未正式进入有效加载回调。
  - 用于覆盖“先卡一下再出进度条”的空窗。
  - 进入 `WebPage` 时立即设为 `true`。
  - 一旦收到首个有效加载信号，例如 `onPageBegin` 或 `onProgressChange(progress > 0)`，立刻关闭。

- `pageLoading`
  - 表示当前网页仍在持续加载过程中。
  - 主要用于驱动顶部细进度条，不再绑定中央全屏遮罩。
  - 在 `onPageEnd` 或主文档错误时结束。

中央加载遮罩的策略调整为：

- 仅在 `bootLoading` 阶段显示。
- 一旦 Web 已开始实际加载，中央遮罩立即消失。
- 后续只保留顶部细进度条作为加载反馈。

这样可以同时解决：

- 进入页面前期空窗。
- 页面可见后中央转圈还压着内容的问题。

### 2. 顶部导航栏视觉

顶部导航保留按钮功能，但调整为“纯操作条”：

- 去掉额外的标题展示和会造成“网页标题栏”联想的层次。
- 去掉调试浮层和调试文字在正式页面中的展示。
- 保持按钮仅包含：
  - 返回
  - 刷新
  - 切换服务器
- `网络自检` 按钮与调试信息一起从正式 UI 中移除，不再面向用户显示。

颜色与层级目标：

- 导航条应更接近 Android 当前工具条的简洁观感。
- 不使用当前这种明显覆盖网页内容的半透明悬浮信息层。

### 3. 顶部导航召回交互

Harmony 的导航隐藏/召回逻辑按 Android 版本靠拢：

- 导航显示时：
  - 页面加载中不自动隐藏。
  - 非加载、非错误、非弹层态时，延迟自动隐藏。

- 导航隐藏时：
  - 顶部保留一条很薄的召回热区。
  - 用户从该区域向下滑动达到较低阈值即可召回。

实现目标：

- 热区更稳定可命中。
- 向下滑动手势更容易成功，不需要明显长距离拖拽。
- 自动隐藏和召回节奏与 Android 当前行为尽量一致。

### 4. 图标同步

Harmony 的 `app_icon.svg` 直接改为 Android 图标同款造型：

- 黑色圆角方形底。
- 白色主图形与白色描边。
- 不再使用蓝色渐变 A 图标。

原则：

- 以 Android 现有 launcher 图标为唯一来源。
- Harmony 只做格式适配，不做样式再设计。

## 风险与处理

### 加载状态判断偏早

- 如果仅根据 `progress > 0` 就移除中央遮罩，极端情况下可能出现页面仍空白但遮罩已消失。
- 处理方式：
  - 允许中央遮罩在 `bootLoading` 阶段极短存在。
  - 一旦进入真实 Web 加载阶段，就优先让网页可见，避免继续遮挡。

### 召回热区过薄导致误触或难触发

- 如果热区过小，仍会出现不易召回的问题。
- 如果热区过大，又会干扰网页顶部点击。
- 处理方式：
  - 保持薄热区，但把向下滑动阈值降低。
  - 重点对齐 Android 的触发手感，而不是单纯放大热区。

### 图标重绘偏差

- SVG 重绘可能和 Android vector 细节不完全一致。
- 处理方式：
  - 以 Android 路径结构为参考，优先保证轮廓、配色和整体识别一致。

## 验收标准

- 进入 WebPage 后，顶部进度反馈能更快出现，不再有明显“先卡住再开始加载”的空窗感。
- 网页首屏内容一旦出现，中央加载遮罩立即消失。
- 页面加载过程中仅保留顶部细进度条，不再让中央转圈持续覆盖网页。
- 顶部不再存在“半透明网页标题栏”的观感。
- 隐藏后的导航栏能通过顶部下滑更容易召回，手感接近 Android。
- 正式页面不再显示调试文字与 `网络自检` 按钮。
- Harmony 应用图标与 Android 当前图标造型一致。
